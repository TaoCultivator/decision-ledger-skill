# Discussion Memo Template

Use this for discussion-only conclusions that are not yet implemented or validated.

```yaml
---
id: short-stable-id
status: proposed
project: project-name
scope: feature-or-domain
module: path.or.component
problem: concise problem statement
symptoms:
  - observed symptom
keywords:
  - exact term
  - synonym or Chinese term
related_files:
  - path/to/spec-or-code
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
---
```

## Context

State what triggered the discussion and what is explicitly in scope.

## Observations

List concrete symptoms, examples, screenshots, samples, or user feedback. Keep facts separate from interpretation.

## Working Agreement

Record the current theory or shared direction. Mark it as unimplemented and unvalidated.

## Risks

List what this approach could break, especially calculations, defaults, migrations, exports, UI behavior, and historical baselines.

## Validation Needed

List what must be checked before this can become an accepted decision.

## Not Decided

List open questions and alternatives that should not be treated as agreed.
